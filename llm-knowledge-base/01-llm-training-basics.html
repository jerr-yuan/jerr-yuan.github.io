LLM 训练基础

从预训练到后训练、蒸馏、微调优化与 Loss——一篇看懂大语言模型的训练全链路，并把每个概念落到自己的项目场景。

1预训练 Pre-training

⏱ 2 分钟

在海量无标注文本上做自监督学习（next-token prediction，"下一个词预测"）。这是 LLM 一切能力的起点。

项目内容
目标学会语言规律、世界知识、推理基础
数据万亿 token 量级（互联网、书籍、代码）
产物base model（基座模型，只会续写、不会聊天）
例子GPT-4 base、Llama 3 base、Qwen base
类比
9 年义务教育 + 高中——通识打底，但还不会干具体工作。

2后训练 Post-training

⏱ 2 分钟

把"会说话"的 base model 训成"能干活、守规矩"的 assistant model。三大阶段：

阶段全名作用
SFTSupervised Fine-tuning用问答对教模型怎么回答
RLHF / DPO偏好对齐让回答符合人类偏好（A 比 B 好）
Safety安全对齐加护栏，拒绝有害请求

数据量小（百万-千万），但 ROI 极高——决定了模型"性格"和"可用性"。

类比
大学专业课 + 入职培训。

3蒸馏 Distillation

⏱ 4 分钟 · 含硬蒸馏 vs 软蒸馏对比

用大模型（teacher）教小模型（student），让小模型逼近大模型表现。目的：压缩、降本、可部署。

按 teacher 给 student 的"信号"形式不同，分硬蒸馏和软蒸馏。

3.1 硬蒸馏 Hard Distillation

Teacher 只给最终答案（argmax 后的具体 token / 类别），Student 学的是"teacher 做了什么决策"。

  • 本质：拿 teacher 生成的数据当 SFT 数据训 student
  • 信息量：少（丢掉了概率分布）
  • 不需要：访问 teacher 的内部 logits → 可以蒸 API（GPT-4、Claude）
  • 典型应用：Alpaca / Vicuna 用 GPT-4 造数据训 LLaMA
类比
师傅只告诉徒弟"这题选 A"。

3.2 软蒸馏 Soft Distillation

Teacher 给完整概率分布（softmax 输出，通常带温度系数 T），Student 学的是"teacher 认为各候选答案的相对概率"。

  • Loss：KL 散度 KL(student ‖ teacher) 或带温度的 cross-entropy
  • 信息量：大（保留 dark knowledge——错误答案之间的相对优劣）
  • 需要：访问 teacher 的 logits → 只能蒸自家或开源模型，蒸不了 API
  • 典型应用：DistilBERT、Qwen 蒸馏版（72B → 7B）、TinyLlama
类比
师傅说"A 60%、B 30%、C 10%，因为 A 和 B 在某种意义上接近"——传授的是判断力的"质感"。

3.3 一表对比

维度硬蒸馏软蒸馏
Teacher 输出最终 token完整概率分布
信息量低高（含 dark knowledge）
需要 logits❌ 不需要✅ 必须
可蒸 API✅ 可以❌ 不行
实操难度低（造数据 + SFT）中（要对齐 vocab、温度系数）
典型场景数据合成（Alpaca/Vicuna）模型压缩（DistilBERT）
业务联系 · mvp-airobot 教练-选手

mvp-airobot 里的 LLM 教练-选手模式严格来说是硬蒸馏的工程变种——不直接训权重，而是用 Claude 当教练改 prompt + bad case 反馈，让 qwen-flash 在推理时表现得更像 Claude。

本质上把 Claude 的判断力通过 prompt 工程而非权重更新 distill 到便宜模型上，所以也叫"prompt 蒸馏 / 行为蒸馏"。这是当前大模型生态里最实用的一类——API 时代蒸不了 logits，但能蒸 prompt 和数据。

4微调表现优化

⏱ 5 分钟 · 四层框架 · 按 ROI 排序

按"先调便宜的、再调贵的"顺序，一共四层。每层 ROI 递减、改动成本递增。

4.1 第一层：数据层（最高 ROI）

90% 的微调失败，是数据问题，不是模型问题。

动作做什么关键洞察
质量 > 数量1K 条精心标注 > 10K 条噪声LIMA 论文：高质量小数据集就能超过大杂烩
Bad case 清洗去标注错误、去重、去模板化样本模型会"忠实地"学会标注员的口头禅和错误
困难样本加权错误率高的样本上采样 2-5 倍比堆数据有用
数据合成用强模型造数据（硬蒸馏）Claude/GPT-4 造 → qwen 学
分布平衡类别、长度、场景分布对齐线上训练分布≠线上分布 = 永远学不会
混入通用数据8:2 或 9:1 配比通用 SFT 数据防止"灾难性遗忘"通用能力

4.2 第二层：训练层

动作关键参数 / 选择
超参调优learning rate 最敏感（1e-5 ~ 5e-5）；epoch 1-3 别多
LoRA vs Full FT数据 < 10K → LoRA；数据 > 100K + 场景集中 → Full FT
训练范式SFT 是基线；行为对齐用 DPO（比 RLHF 简单 10 倍）
课程学习先简单样本、后困难样本，loss 更稳
多任务混合按比例混 batch，不要分阶段（会忘）

4.3 第三层：推理层（不动权重，先压榨干净）

动作何时用
Prompt 优化Few-shot / CoT / 结构化输出（JSON schema）
解码策略temperature 0.0-0.3 任务型 / 0.7+ 创意型；top-p 0.9
RAG 检索增强知识类问题别硬塞进权重，外置向量库 + 检索
结构化约束constrained decoding（guidance / outlines）

4.4 第四层：评估与闭环（最容易被忽略）

动作做什么
Golden Set100-300 条精标测试集，分场景覆盖
离线评估指标业务指标 + LLM-as-Judge 双轨
Bad Case 沉淀错例归类、加权回训练集（迭代飞轮）
A/B 灰度5% → 20% → 50% → 100%
自迭代闭环强模型当教练 → 改 prompt/数据 → 重训/重测

4.5 ROI 速记

层ROI成本见效速度适用阶段
数据⭐⭐⭐⭐⭐低1-3 天永远第一
推理⭐⭐⭐⭐极低几小时微调前必先压榨

5Loss 是什么

⏱ 3 分钟 · 含两条曲线图解

一句话：Loss = 模型预测和真实答案的"差距分"，越小说明模型越接近正确。训练就是在让这个数字下降。

5.1 LLM 里的 Loss 具体是什么

LLM 训练用的是交叉熵损失 Cross-Entropy Loss，本质是"预测下一个 token 时，给真实 token 的概率有多高"。

loss = −log( 模型给真实 token 的概率 )

举例：训练数据是「今天天气真好」，模型在预测最后一个 token 时：

模型给"好"的概率Loss 值含义
0.014.6几乎没猜到，差距巨大
0.12.3差
0.50.69一般
0.90.10很好
1.00完美（但通常做不到，也不健康）

5.2 关键：两条 Loss 曲线

训练时永远要同时盯两条线：

曲线是什么健康状态
Training Loss训练集上的 loss一直下降（不下降 = 训练有 bug）
Validation Loss验证集（模型没见过的数据）上的 loss先降后升 = 过拟合警报
图 5-1 · Training loss 一直降，Validation loss 先降后升——拐点即过拟合开始

5.3 类比（考试场景）

ML 概念考试类比
Training Loss平时刷题的错题数
Validation Loss模拟考的错题数
Loss 下降越练越准
Validation loss 回升平时题做得越来越好，模拟考反而退步 → 开始死记答案，丢了举一反三能力
过拟合学生背了 1000 道题，换张卷子就懵
Early Stopping看到模拟考成绩开始降，立刻停止刷题

5.4 实操数字范围

阶段Training Loss 典型范围
训练刚开始2.0 ~ 3.5
健康收敛后0.5 ~ 1.5 ← 目标区间
< 0.3警惕过拟合
≈ 0100% 背下了训练集，泛化能力归零，必废
实操口诀

• Training loss 不降 → 学习率太小 / 数据有问题 / 模型 cap 不够

• Training loss 降、Validation loss 升 → 过拟合，立即 early stop

• 两条都不降 → 数据噪声太大或学习率太大

业务联系 · 你能不能看到 loss

• qwen-flash prompt 蒸馏（mvp-airobot 教练-选手）→ 看不到 loss，你的"loss"是业务指标（合规率/接通率/bad case 数）

• 如果跑 LoRA 微调 → 每个 step 都会打印 loss，HuggingFace Trainer / LLaMA-Factory 默认有曲线图

• 云厂商托管训练（阿里云百炼调 qwen）→ 控制台有 loss 曲线，上线前必看这张图，曲线形状不对就别上

论文 / 经典

  • LIMA (Meta, 2023) — "Less Is More for Alignment"，证明 1K 高质量样本可超越大杂烩

  • Distilling the Knowledge in a Neural Network (Hinton, 2015) — 知识蒸馏开山之作

工具链

  • HuggingFace Trainer — SFT/DPO 训练标准框架
  • LLaMA-Factory — 中文社区主流微调一站式工具
  • Axolotl — 海外更主流，配置驱动训练
  • TRL — HuggingFace 出的 RL 训练库（含 DPO/PPO）

评估

  • LM Evaluation Harness — 业界标准 benchmark 套件
  • OpenCompass — 中文社区评测平台
