## 主要项目

### LLM 智能巡检 Agent

- **架构**：ReAct 循环（Observe → Think → Act → Observe），最多 12 轮迭代；支持 Claude Opus / Sonnet / Haiku 模型切换
- **工具**：定制工具链——`get_data_overview` / `query_dpd_trend` / `query_repay_trend` / `detect_anomaly` / `query_process_metrics` / `generate_report`
- **输出**：自动生成 McKinsey 风格 HTML 归因报告，覆盖风险线（DPD30/M0）/ 回收线（D1/自然日）/ 过程线（坐席/案量）
- **效果**：归因准确率 > 85%，报告生成 < 3 分钟，人工复核率 < 20%

### AI 催收 Agent

- **架构**：双模型嵌套——Claude Opus 主推理循环 + Sonnet 转写文本分析；工具链：`get_case_info` / `analyze_transcript` / `assess_intent` / `query_strategy_library` / `recommend_action` / `log_decision`
- **Flywheel 机制**：Agent 决策写入日志，还款结果回流验证，持续优化策略库权重
- **技术难点**：催收录音以 Tagalog（菲律宾语）为主，构建多语言意愿识别框架（关键词规则 + 多语言 LLM）
- **输出**：结构化 HTML 评估报告（意愿评分 / 风险因素 / 策略推荐 / 下次跟进时点）

### 触达策略优化与 ROI 分析

- **方法**：AB 测试设计 + 分层回归（控制客群风险差异）+ Dose-Response 弹性拟合 + ICER 增量收益决策
- **发现**：客群分化显著——高风险客群（DPD30≥15%）对 SMS 无弹性，弱组（3条）最优；低风险客群（DPD30≤5%）响应明显，但强组边际收益不足，转向分层精简
- **落地**：分层触达节省 32% 触达成本；叠加 SMS 缩短年化降本 32%–66%
- **交付**：12 张 ApexCharts 可视化 Dashboard，完整方法论文档与 SQL 脚本

- **平台**：Streamlit 多页面 App，结构化报告设计体系，统一 Agent 管理入口
- **Agent 管理**：ReAct 实验室（模型选择 + API Key 配置 + 运行记录）；5 维质检体系；自动化成熟度 OKR 评估
