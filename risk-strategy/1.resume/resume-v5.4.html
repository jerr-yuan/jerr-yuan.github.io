主要项目

LLM 智能巡检 Agent

  • 问题：日常风险指标监控依赖人工，归因维度易遗漏，不同分析师结论一致性低
  • 解决：专家规则（阈值检测 + 已知模式匹配）与 LLM Agent（ReAct 推理归因）双系统架构，结论交叉验证，冲突上报人工复核
  • 效果：替代日常人工看数流程，产出结构化归因报告直接交付业务方；人工仅需复核异常结论

AI 催收 Agent (开发中)

  • 问题：催收人力成本高且优秀话术依赖个人经验，难以规模化复制，新人爬坡周期长
  • 解决：LLM 录音转写 → 意愿识别 → 分客群话术评分与推荐；利用还款结果作为天然标注信号驱动 Flywheel 持续优化
  • 效果：四阶段渐进路径（归因 → 辅助 → Copilot → Agent），Phase 1 已交付话术分析系统，逐步提升 Agent 自主决策能力

触达策略优化与 ROI 分析

  • 问题：SMS 触达强度与逾期回收之间的因果效应未被量化，增发决策缺乏 ROI 依据
  • 解决：AB 测试 + 因果推断（CATE 分客群弹性 + Dose-Response 剂量效应），用 ICER 做增量收益决策
  • 效果：高风险客群 SMS 无弹性（结构性违约），分层精简节省 32% 触达成本；年化降本 $118K–$245K

贷后产品 AI 中台（个人项目）

  • 问题：多 Agent 与策略工具分散运行，缺乏统一管理入口、质量度量和知识沉淀机制

老客额度弹性策略

  • 问题：老客额度策略缺乏量化框架，提额/降额对风险暴露与发标意愿的影响未被系统评估

  • 效果：输出分层差异化额度调幅建议，每个客群附带风险-收益量化依据
