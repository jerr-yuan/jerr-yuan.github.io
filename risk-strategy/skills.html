Core Methodology

The Playbook

策略从 0 到 1 方法论

  • 诊断与定调：痛点与根因梳理，以及用少量关键词锚定战略目标。
  • 机制设计：按业务关键维度做分层与差异化，明确核心抓手与合规边界。
  • 假设验证：用对照实验验证假设，并用 ROI 与盈亏平衡做商业可行性测算。
  • 原子化执行：将战略拆解为可执行、可验收的单一任务。
Execution Standard

数据分析师标准 SOP

  • 需求对齐：管理意图对齐、成功标准量化、所有约束条件的书面化确认。
  • 可行性评估：主从数据源识别、字段可用性检查与数据质量初步评估报告。
  • Baseline 建立：现状探查、基于“现状不满意”的共识定义 Baseline 验证规则。
  • 工程化实现：在关键节点设置评审门禁，确保方案逻辑清晰、可验证、可回溯。
Business Intelligence

业务 Sense 沉淀

  • 成熟度逻辑 (The Maturity Logic)：理解指标滞后性，实施强制窗口控制以避免误读。
  • 三维穿透归因：从横向批次、纵向时间、切片维度做归因，形成可复用的分析框架。
  • 目标原子化：将日目标拆解至小时、金额段、厂商级别，消除管理的“黑盒”问题。
  • 话术与结论模板：将分析结论固化为业务语言，直接驱动决策。
AI Collaboration

风控策略四类工作框架

  • 角色矩阵：明确日常监控、策略开发、效果追踪、临时需求四类场景。
  • 人机协作边界：明确人在目标与规则定义、Agent 在规模化执行与测算中的分工。
  • 上游决策价值：人的不可替代性在于目标定义、归因判断与迭代方向决策。
Rules

与 Agent 协作规则

  • 先讨论再执行：复述需求、给出方案选项、确认细节后再动手，避免返工。
  • 进度与确认：长任务分阶段同步进度。
  • 版本与修改：迭代时新建版本不覆盖；在稳定版上只增不改，保护可追溯性。
  • 验证标准：数据类产出以金标准对账（如差距为 0），脚本执行结果可被用户直接看到。
  • 签名确认：每次回答结尾签名确认，保证规则被遵守。—— Mr.Yuan

Engineering Standards

工程化编码规范体系

  • Python：pyodps 双路径查询（SELECT vs DDL）、Decimal→float 自动转换、脚本暂停规范，确保 Agent 产出代码可直接上生产。
  • SQL（MaxCompute）：CAST AS STRING、子查询禁 LIMIT（ROW_NUMBER 替代）、MAPJOIN hint、DPD 安全分母，消除高频 ParseError。
  • HTML 报告：McKinsey CSS 设计系统（:root 变量 + 双列布局）、ApexCharts 双轴模式、数据注入标准，统一视觉规范。
  • 报告生产流水线：Python→HTML 拼接管道、V{N}.html 版本命名（永不覆盖）、openpyxl 格式化 helper，沉淀自三个生产平台共 30+ 次迭代。

规范让 Agent 产出与人工标准完全对齐，跨会话零解释成本复用。
