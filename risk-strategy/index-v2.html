S0 Reminder Strategy

  • Method: FT Group Analysis
  • Test: z-test for proportions
  • Metric 1: DPD30 rate
  • Metric 2: Reloan rate
  • Decision: Segmented rules
  • Strong > Weak → keep strategy
  • Strong ≈ Weak → reduce cost
  • Strong < Weak → investigate
Core Code
# z-test for proportion comparison import numpy as np def z_test_proportions(p1, n1, p2, n2): p_pool = (p1*n1 + p2*n2) / (n1+n2) se = np.sqrt( p_pool * (1-p_pool) * (1/n1 + 1/n2) ) z = (p1 - p2) / se return z, p_value(z) # * p<0.1 ** p<0.05 *** p<0.01

8-Dimension Quality Scoring Engine
utils/agent_quality.py · Agent Quality Assessment
# Output Stability: CV-based scoring (Agent-friendly) # ReAct agents have high CV by nature: tool_use ~50 tokens vs report ~3000 tokens # CV 1.0-2.0 is NORMAL for agents, should not be penalized def _score_cv(cv: float) -> float: if cv <= 0.5: s = 8 + min((0.5 - cv) * 4, 2) # 9-10: very stable elif cv <= 1.0: s = 6 + (1.0 - cv) * 4 # 6-8: mild variation elif cv <= 2.0: s = 4 + (2.0 - cv) * 2 # 4-6: agent normal range else: s = max(3, 4 - (cv - 2.0) * 1) # 3-4: truly unstable return round(min(s, 10), 1)

Design Principles
PrincipleDescription
Agent-Friendly ThresholdsReAct agents have high CV by nature (tool_use ~50 tokens vs report ~3000 tokens). CV 1.0-2.0 is normal, not penalized.
JSONL Source of TruthAll metrics computed from Claude Code session logs (.jsonl), not self-reported.
Deterministic ScoringSame input always produces same score. No LLM in the scoring loop.
